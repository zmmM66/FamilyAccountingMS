# 家庭记账管理系统登录页设计规范

## 1. Visual Theme & Atmosphere

设计定位为“可信、清晰、有家庭温度的金融工具”。登录页不是营销落地页，而是家庭账本系统的入口，因此视觉应保持安静、专业、信息层级明确。整体氛围采用深色左侧品牌区与浅色表单区的分栏布局，强调账户、权限、审批、家庭协作等核心概念。

关键词：可信、克制、家庭协作、财务清晰、轻量仪表盘。

## 2. Color Palette & Roles

```css
:root {
  --bg: #f4f7f4;
  --panel: #ffffff;
  --panel-soft: #eef5f1;
  --text: #172429;
  --muted: #637177;
  --line: #d8e2df;
  --primary: #0f766e;
  --primary-dark: #0a5d58;
  --primary-rgb: 15, 118, 110;
  --accent: #c4622f;
  --accent-rgb: 196, 98, 47;
  --blue: #2f6fed;
  --danger: #b42318;
  --hero-ink: #eef8f4;
  --hero-muted: #b8cbc8;
}
```

主色用于关键行动按钮和身份标记；橙色用于提醒、待审批和资金流提示；蓝色用于数据说明；红色仅用于错误和拒绝操作。

## 3. Typography Rules

中文优先使用系统中文字体，英文与数字使用系统 UI 字体回退。

```css
font-family: "Microsoft YaHei", "PingFang SC", "Noto Sans SC", "Segoe UI", sans-serif;
```

标题：40-48px，行高 1.12；表单标题和卡片标题：16-22px；正文不低于 15px；说明文字行高不低于 1.7。全站 letter-spacing 保持 0。

## 4. Component Stylings

按钮：8px 圆角，最小高度 44px；主按钮使用绿色实底，悬停加深；次级按钮使用白底边框；危险按钮只在拒绝操作中使用红色文字。

表单：输入框 46px 高，浅色边框，focus 时使用主色阴影。表单卡片使用白底、细边框、柔和阴影。

状态标签：胶囊形，字号 12-13px，用于身份、审批状态、系统能力点。

## 5. Layout Principles

桌面端采用两栏：左侧品牌与系统介绍，右侧操作面板。左侧宽度略小于右侧，避免营销化过强；右侧表单宽度控制在 560px。移动端上下排列，表单优先可读，按钮保持 44px 以上触控高度。

## 6. Depth & Elevation

登录页使用两层深度：

- 页面背景：浅绿灰；
- 操作面板与预览卡片：白底 + 轻阴影；
- 左侧深色区域内部用半透明边框表现数据面板。

阴影保持柔和，不使用厚重浮夸的玻璃拟态。

## 7. Animation & Interaction

动效档位：L1 精致静态。

实现内容：

- 登录页左右区域加载时轻微 fadeIn；
- 按钮 hover 时轻微上浮；
- 表单 focus 时出现主色描边；
- 数据预览卡片使用轻微浮动动画；
- `prefers-reduced-motion` 下关闭动画。

## 8. Do's and Don'ts

Do:

- 优先突出注册、登录、家庭 ID、用户 ID 等关键入口信息。
- 保持金融工具的可信感。
- 使用清晰的审批状态说明。
- 让注册家庭和加入家庭两个选项视觉权重接近。
- 移动端保证按钮和输入框可触摸。
- 错误提示靠近表单。
- 使用家庭账本相关的数据预览增强真实感。
- 使用简洁动效提升质感。

Don't:

- 不使用大面积炫彩渐变或装饰光球。
- 不把登录页做成过度营销化 landing page。
- 不在表单区域堆叠过多说明文字。
- 不使用会干扰阅读的强动效。
- 不让孩子、管理员、创建者权限说明混在一起造成理解负担。
- 不使用游客入口。

## 9. Responsive Behavior

断点：900px。

桌面端双栏布局，左侧介绍与右侧入口并排；移动端改为单栏，左侧介绍缩短，右侧面板铺满宽度。所有可交互元素最小高度 44px，防止移动端误触。表单双列字段在移动端折叠为单列。
